بص يا باشا، هجوم DHCP Starving هدفه إنك تشلّ سيرفر الـ DHCP بحيث يوزّع كل الـ IPs اللي عنده، ووقتها الضحايا الجداد ما يعرفوش ياخدوا IP، وساعتها تقدر أنت تركب مكانه سيرفر مزيف (Rogue DHCP) وتتحكم فيهم.

الفكرة ببساطة

الـ DHCP Server بيكون ليه pool محدود من الـ IPs (مثلاً 192.168.1.2 → 192.168.1.254).

الهاكر يبعث requests وهمية كتير جدًا كأنه أجهزة كتير بتطلب IP.

السيرفر يوزع كل الـ IPs اللي عنده ويخلص.

الضحايا مش هيلاقوا IP، وساعتها الهاكر يقدر ينصب DHCP server مزيف ويوزع عليهم إعدادات هو عايزها (زي Gateway وهمي أو DNS وهمي).

الخطوات العملية
1️⃣ تأكد إن الشبكة فيها DHCP

شوف إذا جهازك بياخد IP أوتوماتيك ولا لأ:

ip a


أو جرب تعمل:

sudo dhclient -v

2️⃣ اعمل الهجوم (Starving)

أشهر أداة: yersinia
تشغّلها كده:

sudo yersinia -I


هتفتح واجهة interactive.

تختار DHCP.

بعدين تختار الهجوم:

attack 1


الأداة هتبعت طلبات كتير لحد ما السيرفر يخلص الـ pool.

3️⃣ شغّل Rogue DHCP (اختياري)

لو عايز تتحكم في الضحايا:

استخدم dhcpd أو أداة زي metasploit auxiliary/server/dhcp.

وزع عليهم Gateway و DNS أنت محددهم.

تقدر تعمل MITM (Man in the Middle) أو Sniffing بسهولة.

أدوات تانية مفيدة

dhcpstarv: أداة بسيطة للهجوم.

sudo dhcpstarv -i eth0


Metasploit module:

use auxiliary/dos/dhcp/dhcp_starvation

DHCP Snooping
